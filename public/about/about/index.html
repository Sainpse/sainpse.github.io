Describe your experience building large systems with many services - web front ends, REST APIs, data stores, event processing and other kinds of integration between components. What are the key things to think about in regard to architecture, maintainability, and reliability in these large systems?

The first thing that comes in mind regarding my experience building large systems is microservices architecture, agility, modularizatrion and continues delivery.

Microservices Architecture (Lego Blocks) This architecture is well suited for large systems because it allows a team to build the system as well defined modules which are easy to understand, maintain, independently deploy and reuse in other similar projects. I think of it as lego blocks which can be rearranged for different programs.

In my previous role we used openshift to deploy our microservices which is a relatively easier platform for the kubernetes cluster. The micorservice allowed me and my team to quickly isolate and fix issues without any downtime for the entire system. Furthermore this allowed me to utilize diferent programming languages strength for certain components.

REST APIs for Communication between the microservices In my current and past positions the rest API approach has been a good communication layer to intergrate different specialized components. Its stateless property improves the performance and reliablity because we do not have to deal with server-side state synchronization logic. Further more the layered architecture for rest api’s allows us to place a proxy or load balancer btween the client and the server and thus improving scalability. Finally using JSON web tokens to allow secure communication between components

Data Stores One of the key things to also consider is the type of format, volume, velocity and feature set for data requirments for the system in question. For instance on one of my large projects we had data that required high intergrity and relationships to be strictly enforced using DB constraints which required us to use a relational database like postgress. On the other hand for intergrating are a portion of complex data structures that evolve we used MongoDB to maintain a high-performing system and agility. This is why the microservice architecture is well suited for large system, given it can support a hybrid dat model of any tech stack to archieve a goal effeciently.

DevOps This is crucial expecialy to maintain order, synchronization and continouse delivery between different teams working on different services for the system. Each component release must be evaluated and impact analysis needs to be done on every other component that depends on it. Additionaly monitoring tools like grafana which I have worked with need to be incorporated for continuous feedback on speed and failures for further improvements.

Web frontends Although my exprience is mostly has on the backend, data and the anaytics layer I have heard my fair share of web apps development. I know the UI/UX has to be intuitive and fast. Hence all having data processing should be done in the backend and served to the frontend. the technology frameworks I have used is react, redux for session management and your typical javascript, HTML and CSS for styling.
